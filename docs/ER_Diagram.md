# Entity Relationship Diagram (ER Diagram)

## Description

The Entity Relationship Diagram (ERD) represents the logical database structure of the Voice-Based Concept Understanding Analyser (VBCUA). It defines entities, attributes, and relationships required for storing and processing application data.

## Main Entities

* User
* Audio File
* Transcript
* Audio Feature
* Evaluation Result
* Reference Concept
* Semantic Similarity
* Report
* Session

## Key Relationships

* Users upload audio files
* Audio files generate transcripts
* Transcripts are analyzed for semantic similarity
* Audio features are extracted from uploaded audio
* Evaluation results generate final scores and reports

## System Purpose

The ER diagram supports audio upload, speech transcription, semantic analysis, understanding assessment, report generation, and session tracking.
